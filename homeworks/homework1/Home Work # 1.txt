Home Work # 1

* VCS
Завести аккаунт на github, при выполнении домашнего задания продемонстрировать умение работать с системой контроля версий, иметь настроенное имя и эл.почту в конфигурации, внести в .gitignore все неотносящиеся к логике проекта файлы. Продемонстрировать разумное разбиение выполняемой работы на коммиты.

* Основы разработки
Напишите приложение, которое будет содержать методы:
1. Преобразования десятичного числа в двоичное
2. Преобразования десятичного числа в шестнадцатеричное
3. Преобразования двоичного числа в восьмеричное
4. Валидации входной строки - является ли числом одной из системы исчислений (двоичная или восьмеричная или шестнадцатеричная). Если является - возвращать true.
Методы #1,2,3 возвращают число в формате String.
Метод #4 возвращает boolean.
Велосипеды не изобретаем, стараемся по максимуму использовать стандартные возможности/библиотеки jdk для конвертации чисел.

* TestNG
Покройте каждый из методов двумя или более юнит тестами (помним про параметр expectedExceptions для негативных сценариев, а так же про возможнось параметризации тестов через @DataProvider). Для каждого из тестов задать описание @Test(description=""). Тесты должны запускаться параллельно, в 5 потоков. Необходимо создать конфигурацию testng.xml, с выделением нескольких блоков <test>. Перед каждым тестовым методом должен выводиться id текущего потока. (Thread.currentThread().getId();)

* Maven
Домашнее задание должно быть выполнено с использованием системы сборки maven. Тесты проекта должны иметь возможность запускаться из командной строки командой "mvn test"
